# Back-End-API
API для разработки приложения на android ios
# **RESTful API Документация**

Добро пожаловать в документацию для RESTful API бекенда мобильного приложения. Этот API предоставляет доступ к функциям авторизации, управлению профилем, истории пользователя и данным по категориям.

## **Аутентификация**

### **Вход в систему (`POST /auth/login`)**

Аутентификация пользователя. Возвращает токен доступа.

**Параметры запроса:**

- **`username`**: Имя пользователя
- **`password`**: Пароль пользователя

**Пример запроса:**

```bash

curl -X POST -d "username=user&password=pass" http://localhost:8080/auth/login

```

**Обработка ошибок:**

- **`400 Bad Request`**: Некорректный запрос (например, отсутствие обязательных параметров)
- **`401 Unauthorized`**: Неверные учетные данные

**Логирование ошибок:**

Логируются некорректные запросы и ошибки аутентификации.

### **Выход из системы (`POST /auth/logout`)**

Завершение сеанса пользователя. Требует наличие действующего токена доступа.

**Пример запроса:**

```bash

curl -X POST -H "Authorization: Bearer your_access_token" http://localhost:8080/auth/logout

```

**Обработка ошибок:**

- **`401 Unauthorized`**: Недействительный токен доступа

**Логирование ошибок:**

Логируются ошибки, связанные с недействительным токеном доступа.

## **Профиль**

### **Получение профиля (`GET /profile/get`)**

Возвращает информацию о текущем пользователе. Требует наличие действующего токена доступа.

**Пример запроса:**

```bash

curl -X GET -H "Authorization: Bearer your_access_token" http://localhost:8080/profile/get

```

**Обработка ошибок:**

- **`401 Unauthorized`**: Недействительный токен доступа
- **`404 Not Found`**: Профиль пользователя не найден

**Логирование ошибок:**

Логируются ошибки, связанные с отсутствием профиля пользователя.

### **Обновление профиля (`PUT /profile/update`)**

Обновление данных профиля текущего пользователя. Требует наличие действующего токена доступа.

**Параметры запроса:**

- **`name`**: Новое имя пользователя

**Пример запроса:**

```bash

curl -X PUT -H "Authorization: Bearer your_access_token" -d "name=new_name" http://localhost:8080/profile/update

```

**Обработка ошибок:**

- **`401 Unauthorized`**: Недействительный токен доступа
- **`400 Bad Request`**: Некорректный запрос (например, отсутствие обязательных параметров)

**Логирование ошибок:**

Логируются ошибки, связанные с недействительным токеном доступа и некорректными запросами.

## **Аутентификация через API-ключ**

Для всех запросов требуется наличие заголовка **`API-Key`** с действующим API-ключом.

**Пример запроса с API-ключом:**

```bash

curl -H "API-Key: your_api_key" http://localhost:8080/auth/login

```

Пожалуйста, замените **`your_access_token`** и **`your_api_key`** на актуальные значения.

**Логирование ошибок:**

Логируются попытки запросов с недействительными API-ключами.

## **Логирование**

Весь back-end ведет журнал (лог) событий. Логи содержат информацию о каждом запросе, включая успешные и неуспешные запросы, а также информацию об ошибках.

Логи сохраняются в файлы для последующего анализа и мониторинга работы системы.

Помимо обработки ошибок, также логируются другие важные события, такие как успешные операции, изменения в системе и действия пользователей.
